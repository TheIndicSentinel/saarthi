# First signed Kisan pack release — runbook

This publishes the first **signed** Kisan pack so the in-app update channel
(`PackUpdateWorker`) has something real to pull. The app verifies SHA-256 +
ECDSA P-256 signature against the public key embedded in
`app/build.gradle.kts` (`KISAN_PACK_PUBLIC_KEY`) before installing.

Artifacts in this folder:
- `kisan_pack.json` — the v2 pack (34 entries, `packVersion: 8`). **These exact
  bytes get signed and uploaded — do not reformat after signing.**
- `make-manifest.sh` — computes SHA-256, signs with your offline key, writes
  `manifest.json`, and self-verifies.

## 0. One-time: confirm the key pair matches the app
The signature is useless unless your private key matches the public key baked
into the app. Verify they are the same key:

```bash
openssl ec -in .keys/saarthi-pack-private.pem -pubout -outform DER | base64
```
The output MUST equal `KISAN_PACK_PUBLIC_KEY` in `app/build.gradle.kts`:
```
MFkwEwYHKoZIzj0CAQYIKoZIzj0DAQcDQgAET92OpbPOpjrzlo6eOURyOtsbxfpBPElsQcCo8hce3VBfpKLTkCr7szYyMiifJE/Mko/ZCwqvqFOpk7hfdyia9Q==
```
If they differ, stop — re-sign with the right key or update the app's key.

## 1. Prereqs
- `gh` (GitHub CLI) authenticated with push access to `TheIndicSentinel/saarthi-packs`.
- `openssl`, `python3`.
- The public repo `saarthi-packs` exists.

## 2. Sign + build the manifest
The asset URL is deterministic (tag `kisan-v8`), so we can compute the manifest
before creating the release:

```bash
cd docs/packs-release
./make-manifest.sh kisan_pack.json ../../.keys/saarthi-pack-private.pem \
  https://github.com/TheIndicSentinel/saarthi-packs/releases/download/kisan-v8/kisan_pack.json \
  30
```
- The last arg `30` is `minAppVersionCode` — set it to the `versionCode` of the
  **first app build that contains the v2 parser** (the P1 changes). If you bump
  the app's `versionCode` for that release, use the new number here.
- Expect `signature self-check: OK`.

This writes `manifest.json` next to `kisan_pack.json`.

## 3. Create the GitHub Release (uploads both assets)
```bash
gh release create kisan-v8 \
  docs/packs-release/kisan_pack.json \
  docs/packs-release/manifest.json \
  --repo TheIndicSentinel/saarthi-packs \
  --title "Kisan pack v8" \
  --notes "First signed Kisan knowledge pack (34 entries). Signed with ECDSA P-256."
```
GitHub then serves the manifest at the stable URL the app polls:
`https://github.com/TheIndicSentinel/saarthi-packs/releases/latest/download/manifest.json`

## 4. Verify the channel end-to-end
```bash
# Manifest resolves and is valid JSON:
curl -sL https://github.com/TheIndicSentinel/saarthi-packs/releases/latest/download/manifest.json | python3 -m json.tool

# Pack asset resolves (expect HTTP 200 after redirects):
curl -sLI https://github.com/TheIndicSentinel/saarthi-packs/releases/download/kisan-v8/kisan_pack.json | head -1

# Offline signature check against the app's embedded PUBLIC key:
APP_PUB=MFkwEwYHKoZIzj0CAQYIKoZIzj0DAQcDQgAET92OpbPOpjrzlo6eOURyOtsbxfpBPElsQcCo8hce3VBfpKLTkCr7szYyMiifJE/Mko/ZCwqvqFOpk7hfdyia9Q==
printf '%s' "$APP_PUB" | base64 -d > /tmp/app_pub.der
SIG=$(curl -sL .../releases/latest/download/manifest.json | python3 -c "import json,sys;print(json.load(sys.stdin)['pack']['signature'])")
printf '%s' "$SIG" | base64 -d > /tmp/sig.bin
openssl dgst -sha256 -verify <(openssl pkey -pubin -inform DER -in /tmp/app_pub.der) \
  -signature /tmp/sig.bin docs/packs-release/kisan_pack.json   # expect: Verified OK
```

## 5. Confirm on device
Install the P1 app build (with the v2 parser + verifier). On Wi-Fi, within the
24h cycle (or trigger sooner in dev), `saarthi_debug.log` should show:
```
[PACK] Kisan pack update available: v7 → v8; downloading …
[PACK] Kisan pack v8 (en, 34 entries) installed from manifest:v8
```
A tampered pack instead logs `Kisan pack REJECTED — invalid signature` and the
old pack is kept.

## Future releases
Bump `packVersion` in `kisan_pack.json` (and any changed `sections[].version`),
re-run step 2 with a new tag (`kisan-v9`, updated asset URL), and step 3. The
app installs only when `packVersion` > the installed version.

## Notes
- The `manifest.json` is generated by the script — do not edit it by hand
  (the signature is over the pack bytes, not the manifest).
- These files currently live in the app repo for the first manual release; the
  long-term home is the `saarthi-packs` repo (with the CI pipeline). Move them
  there when you set that up.
